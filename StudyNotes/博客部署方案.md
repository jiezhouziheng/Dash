**Codex 的方案，明天细看**

下面给你一套最稳的方案：**服务器只负责用 Nginx 托管 Hexo 生成后的静态文件**，Hexo 构建可以在本地或 GitHub Actions 里完成。这样服务器不需要长期运行 Node 进程，也不需要跑 `hexo server`。

**总体方案**

- Hexo 源码仍放 GitHub。
- 每次发布时执行 `hexo clean && hexo generate`。
- 将生成的 `public/` 目录同步到服务器 `/var/www/hexo-blog/`。
- Ubuntu 24.04.2 上用 Nginx 直接通过 `http://服务器IP/` 访问。
- 后续绑定域名后，再改 Nginx `server_name` 并配置 HTTPS。

---

## 1. 服务器初始化

先 SSH 到服务器：

```bash
ssh root@<SERVER_IP>
```

更新系统并安装 Nginx、Git、rsync：

```bash
apt update
apt upgrade -y
apt install -y nginx git rsync ufw
systemctl enable --now nginx
```

创建部署用户：

```bash
adduser deploy
usermod -aG sudo deploy
```

配置防火墙：

```bash
ufw allow OpenSSH
ufw allow 80/tcp
ufw enable
ufw status
```

如果你的云厂商还有“安全组 / 防火墙规则”，也要放行入站 `80/tcp`。

---

## 2. 创建网站目录

```bash
mkdir -p /var/www/hexo-blog
chown -R deploy:deploy /var/www/hexo-blog
chmod -R 755 /var/www/hexo-blog
```

---

## 3. 配置 Nginx

创建站点配置：

```bash
nano /etc/nginx/sites-available/hexo-blog
```

写入：

```nginx
server {
    listen 80 default_server;
    server_name _;

    root /var/www/hexo-blog;
    index index.html;

    access_log /var/log/nginx/hexo-blog.access.log;
    error_log /var/log/nginx/hexo-blog.error.log;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~* \.(css|js|jpg|jpeg|png|gif|svg|ico|webp|avif|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, max-age=2592000";
        try_files $uri =404;
    }
}
```

启用配置：

```bash
ln -sfn /etc/nginx/sites-available/hexo-blog /etc/nginx/sites-enabled/hexo-blog
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx
```

现在访问：

```text
http://<SERVER_IP>/
```

因为还没上传 Hexo 文件，可能会看到 403 或 404，这是正常的。

---

## 4. 修改 Hexo 配置

如果你原来 `_config.yml` 里是 GitHub Pages 地址，例如：

```yml
url: https://username.github.io
```

服务器 IP 访问时建议单独建一个 VPS 配置文件：

```bash
touch _config.vps.yml
```

内容：

```yml
url: http://<SERVER_IP>
root: /
```

这样 GitHub Pages 和 VPS 可以共用同一套源码，但构建时使用不同配置。

本地构建 VPS 版本：

```bash
npm install
npx hexo --config _config.yml,_config.vps.yml clean
npx hexo --config _config.yml,_config.vps.yml generate
```

Hexo 生成结果会在 `public/` 目录。

---

## 5. 手动部署到服务器

在本地博客项目目录执行：

```bash
rsync -az --delete --chmod=D755,F644 public/ deploy@<SERVER_IP>:/var/www/hexo-blog/
```

然后浏览器打开：

```text
http://<SERVER_IP>/
```

如果页面 CSS 丢失，优先检查 `_config.vps.yml` 里的 `url` 和 `root`，以及 Nginx 目录权限。

---

## 6. 推荐：用 GitHub Actions 自动部署

在服务器上确认 `deploy` 用户可以 SSH 登录，并且 `/var/www/hexo-blog` 可写。

本地生成一对部署密钥：

```bash
ssh-keygen -t ed25519 -C "github-actions-hexo-deploy" -f hexo_vps_deploy
```

把公钥放到服务器：

```bash
ssh-copy-id -i hexo_vps_deploy.pub deploy@<SERVER_IP>
```

在 GitHub 仓库 `Settings -> Secrets and variables -> Actions` 添加：

```text
VPS_HOST=<SERVER_IP>
VPS_USER=deploy
VPS_PORT=22
VPS_SSH_KEY=<hexo_vps_deploy 私钥内容>
```

新建 `.github/workflows/deploy-vps.yml`：

```yml
name: Deploy Hexo to VPS

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-24.04

    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive

      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: npm

      - run: npm ci

      - run: |
          npx hexo --config _config.yml,_config.vps.yml clean
          npx hexo --config _config.yml,_config.vps.yml generate

      - name: Prepare SSH
        run: |
          mkdir -p ~/.ssh
          printf "%s\n" "${{ secrets.VPS_SSH_KEY }}" > ~/.ssh/deploy_key
          chmod 600 ~/.ssh/deploy_key
          ssh-keyscan -p "${{ secrets.VPS_PORT }}" -H "${{ secrets.VPS_HOST }}" >> ~/.ssh/known_hosts

      - name: Upload
        run: |
          rsync -az --delete --chmod=D755,F644 \
            -e "ssh -i ~/.ssh/deploy_key -p ${{ secrets.VPS_PORT }}" \
            public/ \
            "${{ secrets.VPS_USER }}@${{ secrets.VPS_HOST }}:/var/www/hexo-blog/"
```

之后你只要 `git push` 到 `main`，GitHub Actions 就会自动构建并发布到服务器。

---

## 7. 后续绑定域名时

等你有域名后：

1. 在 DNS 添加 `A` 记录指向服务器 IP。
2. 修改 Hexo 配置：

```yml
url: https://example.com
root: /
```

3. 修改 Nginx：

```nginx
server_name example.com www.example.com;
```

4. 申请 HTTPS 证书。

现在是 2026 年，Let’s Encrypt 已支持 IP 地址证书，但 IP 证书是 6 天短证书，并且 Certbot 对 Nginx 的自动安装支持还不如域名证书省心。个人博客长期运行仍建议绑定域名后再走标准 HTTPS。

参考官方文档：Hexo 生成静态文件在 `public/`，`hexo clean` 会清理 `public/`；Ubuntu 推荐用 `apt` 管理包；Ubuntu 官方也给出 Nginx 配置方式；GitHub 支持用部署密钥做自动部署；Let’s Encrypt 已公开支持 IP 地址证书但有短证书限制。
